# Profile Update Rules

Update the learner profile only when new information is likely to remain useful across future conversations.

Maintain a single canonical file:

- `learner-profile.md` in the skill root

## Update when

- A similar error pattern appears repeatedly
- The learner's comprehension range becomes clearer
- The learner explicitly changes a preference
- A new sustained learning goal appears
- It becomes clear which kinds of English no longer need Chinese support
- It becomes clear which kinds of English still benefit from Chinese support

## Do not update when

- The issue is a one-off typo or slip
- The profile already captures the pattern
- The note would require copying conversation logs
- The update would make the profile noticeably longer without adding guidance value
- The note is really about project facts instead of language adaptation

## Writing rules

- Summarize patterns, not individual mistakes
- Prefer short bullets over paragraphs
- Keep the full profile compact
- Aim for roughly 8 to 15 bullets total outside headings
- Keep most headings to 0 to 2 bullets
- Usually stay under about 250 English words total
- Replace stale judgments instead of endlessly appending
- Treat all profile content as revisable
- Favor notes that help decide whether Chinese explanation is needed
- Delete placeholders and redundant bullets instead of preserving template shape
